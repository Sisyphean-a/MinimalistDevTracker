# Technical Specification: Minimalist Dev Tracker (VS Code Extension)

## 1. Project Overview

This project is a lightweight, privacy-first, local-only Visual Studio Code extension. Its purpose is to silently track active coding time and effective Lines of Code (LOC) produced, specifically optimized to account for AI-generated code and completely ignore bulk repository updates like `git pull` or branch checkouts. The extension strictly monitors user-configured projects and automatically discovers and tracks their associated Git worktrees. Data is stored entirely locally without requiring external servers or logins.

## 2. Core Architecture: The "Dual-Track" System

The extension separates time tracking and LOC tracking into two distinct mechanisms to ensure high accuracy and low resource consumption.

- **Track 1: Time Flow (Editor Activity)**: Relies on native VS Code event listeners to determine user or AI activity within the editor. Uses a state machine with a 2-minute debounce timer.
- **Track 2: Code Flow (Git Diff State)**: Relies exclusively on Git state (via the built-in VS Code Git extension API or lightweight child processes) to calculate actual code output. This inherently ignores `git pull` operations (which update `HEAD` and leave the working directory clean) while perfectly capturing AI-injected code (which modifies the working directory relative to `HEAD`).

## 3. Feature Requirements & Logic

### 3.1. Project Whitelist & Git Worktree Discovery

- **Configuration**: Provide a setting in `package.json` (e.g., `minimalTracker.trackedPaths`, type: `array` of `string`) where users can define absolute paths to root repository directories.
- **Initialization & Worktree Resolution**:
  - On extension activation, read the `trackedPaths`.
  - For each path, verify it is a valid Git repository.
  - Execute `git worktree list` (or use the VS Code Git API) to discover all active worktrees associated with the root repository.
  - Maintain an active in-memory Set/List of allowed paths (Root Repos + All Discovered Worktrees).
- **Path Validation**: Ignore any VS Code activity if the current file's absolute path does not fall under one of the allowed paths.

### 3.2. Time Tracking State Machine

- **Events**: Listen to `vscode.workspace.onDidChangeTextDocument`, `vscode.window.onDidChangeActiveTextEditor`, and `vscode.window.onDidChangeTextEditorSelection`.
- **State Machine**:
  - **State: IDLE (Default)**.
  - **Action**: When a tracked event occurs in an allowed path, transition to **ACTIVE**. Record `sessionStartTime`.
  - **Debounce/Heartbeat**: Every subsequent event resets a `120-second` (2-minute) countdown timer.
  - **Timeout**: If the timer reaches 0, transition back to **IDLE**.
  - **Session Finalization**: Calculate `sessionDuration = Date.now() - sessionStartTime - 120_000ms`. Log the session.

### 3.3. LOC Tracking Logic (Git-Based)

Do NOT calculate LOC by counting characters or lines injected per keystroke.

- **Mechanism**: Utilize the built-in VS Code Git extension (`vscode.extensions.getExtension('vscode.git')`) to read the repository state, or spawn a lightweight `git diff HEAD --shortstat` process.
- **Snapshot Algorithm**:
  1. When a session transitions from IDLE to ACTIVE, capture the current uncommitted LOC state (Insertions and Deletions relative to HEAD). Call this `baselineDiff`.
  2. When the session transitions from ACTIVE to IDLE, capture the uncommitted LOC state again. Call this `endDiff`.
  3. The LOC produced during this specific session is calculated as `endDiff - baselineDiff`.
- **Handling `git commit`**: Listen to repository state changes. If a commit occurs during an ACTIVE session, the uncommitted diff resets to 0. The extension must detect this, finalize the current LOC chunk immediately before the commit, and reset the `baselineDiff` to 0 to resume counting subsequent edits.

### 3.4. Local Storage & Data Persistence

- **Storage Location**: Use `context.globalStorageUri` to ensure data persists across different workspaces but remains strictly local.
- **File Structure**: Store logs in daily JSON files (e.g., `YYYY-MM-DD.json`). Append data at the end of every ACTIVE session to prevent data loss in case VS Code is closed abruptly.

## 5. Development Directives for AI Assistant

1. **Git Integration**: Prioritize using the VS Code Built-in Git Extension API. To access it:

   TypeScript

   ```
   const gitExtension = vscode.extensions.getExtension('vscode.git');
   const git = gitExtension.exports.getAPI(1);
   ```

   Subscribe to `git.onDidOpenRepository` and `repo.state.onDidChange` to reliably detect commits and repository state shifts.

2. **Performance Constraints**: Ensure the 120-second debounce timer does not cause memory leaks. Clear old timeouts when resetting the heartbeat.

3. **Action Item**: Begin by generating the `package.json` with the required configuration contributions, followed by the core `extension.ts` implementing the Time Tracking State Machine (Section 3.2).
